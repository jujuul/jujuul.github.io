ES6中的Symbol

2020-08-11 2 min read

ES6中引入了一种新的基础数据类型：Symbol，它的功能类似于一种标识唯一性的ID。

let s1 = Symbol()
let s2 = Symbol('another symbol')
let s3 = Symbol('another symbol')
console.log(s1 == s2)   //false
console.log(s1 === s2)  //false
console.log(s2 == s3)   //false
console.log(s2 === s3)  //false
console.log(s1 == s1)   //true
console.log(s1 === s1)  //true
console.log(s2 === s2)  //true

它的值具有唯一性，除了自己它谁都不等于。
console.log(typeof s1) //symbol
typeof的返回值为symbol

let obj = {
            [Symbol('name')]: '学习Symbol',
            age: 18,
            sex: 'male'
        }
        console.log(Object.keys(obj)) // ['age', 'sex']
        for (let p in obj) {
            console.log(p) // 分别输出：'age' 和 'sex'
        }

不能通过Object.keys()或者for...in来枚举
使用JSON.stringify()将对象转换成JSON字符串的时候，Symbol属性也会被排除在输出内容之外：
console.log(JSON.stringify(obj)) // {"age":18,"sex":"male"}
我们就没办法获取以Symbol方式定义的对象属性了么？非也。还是会有一些专门针对Symbol的API，比如：

// 使用Object的API
Object.getOwnPropertySymbols(obj) // [Symbol(name)]
// 使用新增的反射API
Reflect.ownKeys(obj) // [Symbol(name), 'age', 'title']
